📝 Авторський Блог: МОСТОВИЙ Віктор

Авторський Блог — це сучасний, легкий та функціональний веб-додаток (Single Page Application), розроблений для ведення персонального онлайн-журналу. Сайт поєднує в собі мінімалістичний дизайн у стилі Glassmorphism та потужні хмарні технології для зберігання контенту.

🎯 Призначення сайту

Основна мета проекту — надати автору зручний інструмент для публікації думок, технічних статей, кулінарних рецептів та нотаток про хобі. Сайт орієнтований на:

Персональний брендинг: Створення власного простору в мережі.

Швидку публікацію: Можливість додавати контент з будь-якого пристрою в реальному часі.

Інтерактивність: Користувачі можуть читати статті, а автор — керувати ними через захищену адмін-панель.

🏗️ Структура сайту

Сайт складається з чотирьох основних візуальних блоків (екранів), між якими реалізована плавна навігація без перезавантаження сторінки:

Головна сторінка (Стрічка думок):

Вітальний заголовок (Hero section).

Сітка карток статей з категоріями, датами та ім'ям автора.

Творча студія (Адмін-панель):

Форма створення/редагування статей.

Список усіх публікацій з кнопками швидкого редагування та видалення.

Сторінка статті (Detail View):

Повнотекстовий перегляд контенту з фокусом на типографіку та зручність читання.

Система автентифікації:

Екрани входу (Login) та первинного налаштування (Setup) профілю автора.

🛠️ Функціональні можливості

1. Керування контентом (CRUD)

Створення: Написання статей з вибором однієї з п'яти категорій (Життя, Кулінарія, Код, Технології, Хобі).

Читання: Публічний доступ до всіх текстів для звичайних користувачів.

Редагування: Можливість миттєво виправити помилки або оновити інформацію в існуючих статтях.

Видалення: Повне видалення неактуального контенту з підтвердженням дії.

2. Система Ролей

Admin (Адміністратор): Повний доступ до "Творчої студії". Тільки адмін може змінювати базу даних.

User (Користувач): Автоматична роль при реєстрації. Дозволяє переглядати контент, але обмежує доступ до інструментів редагування.

3. Технологічні особливості

Cloud Database (Firestore): Всі статті зберігаються у хмарі Google Firebase. Зміни з'являються у всіх відвідувачів миттєво завдяки технології onSnapshot.

Responsive Design: Сайт адаптований під мобільні телефони, планшети та десктопи.

Анонімна авторизація: Використання Firebase Auth для безпечного з'єднання з базою даних.

💻 Стек технологій

Frontend: HTML5, JavaScript (ES Modules).

Стилізація: Tailwind CSS (для сучасного UI та адаптивності).

Іконки: Font Awesome 6.

Backend/Database: Google Firebase (Firestore для даних, Auth для безпеки).

Шрифти: Plus Jakarta Sans (Google Fonts).

🚀 Як це працює (Логіка)

Перший запуск: Користувач потрапляє на сторінку реєстрації, створює профіль (ім'я + пароль). В базі створюється документ з роллю user.

Активація Адмінки: Власник проекту змінює роль у консолі Firebase на admin.

Публікація: Адмін заходить у "Студію", пише текст. Firebase автоматично додає мітку часу (serverTimestamp) та ім'я автора.

Відображення: Головна сторінка автоматично оновлюється, показуючи нову картку статті у стрічці.
